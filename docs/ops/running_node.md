# راهنمای اپراتور نود Nexus (Node Operator Guide)

این راهنما برای کسانی است که می‌خواهند یک **Super Node** را در شبکه Nexus DePIN راه‌اندازی و مدیریت کنند. به عنوان یک اپراتور، شما با ارائه منابع محاسباتی (CPU/GPU)، فضای ذخیره‌سازی و پهنای باند، در پاداش‌های شبکه سهیم می‌شوید.

---

## ۱. پیش‌نیازهای سخت‌افزاری

برای اجرای یک سوپر نود پایدار که بتواند بارهای کاری هوش مصنوعی (مانند Unsloth) را هندل کند، مشخصات زیر پیشنهاد می‌شود:

| منبع | حداقل (تست) | پیشنهادی (تولید) |
| :--- | :--- | :--- |
| **پردازنده (CPU)** | 8 Cores | 32+ Cores (AMD EPYC / Intel Xeon) |
| **رم (RAM)** | 32 GB | 128 GB+ ECC |
| **کارت گرافیک (GPU)** | RTX 3060 (12GB) | NVIDIA A100 / H100 / 2x RTX 4090 |
| **فضای ذخیره سازی** | 1TB NVMe | 4TB+ NVMe (RAID 1) |
| **اینترنت** | 100 Mbps | 1 Gbps+ (Symmetric) |

---

## ۲. استراتژی‌های استقرار (Deployment)

### الف. اجرای محلی / سرور خانگی (Home Server)
مناسب برای اپراتورهای کوچک که می‌خواهند هزینه‌ها را به حداقل برسانند.
*   **نکته**: حتماً از سیستم خنک‌کننده مناسب برای GPUها استفاده کنید.
*   **شبکه**: نیاز به Static IP یا استفاده از سرویس‌های DDNS و باز کردن پورت‌های `4001` (libp2p) و `443`.

### ب. اجرای ابری (Cloud / Bare Metal)
پیشنهاد می‌شود از سرویس‌دهندگانی که اجازه دسترسی مستقیم به GPU را می‌دهند استفاده کنید (مانند Lambda Labs، Vast.ai یا سرورهای اختصاصی Hetzner).

---

## ۳. مراحل نصب و راه‌اندازی

### ۱. نصب پیش‌نیازهای سیستم‌عامل
نکسوس روی **Ubuntu 22.04 LTS** بهینه شده است.
```bash
sudo apt update && sudo apt upgrade -y
sudo apt install docker.io docker-compose-v2 nvidia-container-toolkit -y
```

### ۲. دریافت و پیکربندی نود
```bash
git clone https://github.com/nexus/super-node.git
cd super-node
cp config/config.example.yaml config/config.yaml
```
ویرایش فایل `config.yaml`:
*   `operator_address`: آدرس کیف پول شما برای دریافت پاداش.
*   `storage_path`: مسیری که داده‌های IPFS و TiDB ذخیره می‌شوند.

### ۳. تولید هویت نود (Identity)
```bash
./nexus-cli keygen
```
**هشدار**: فایل `data/keystore/node.key` را در جایی امن کپی کنید. اگر این کلید را گم کنید، دسترسی به اعتبار و استیک‌های خود را از دست می‌دهید.

---

## ۴. مانیتورینگ و هشدار (Monitoring)

ما از پشته **Prometheus + Grafana** برای نظارت استفاده می‌کنیم.
*   **داشبورد اصلی**: وضعیت سلامت کانتینرها، مصرف GPU و تأخیر Redpanda.
*   **هشدارها (Alerts)**:
    *   اگر دمای GPU از ۸۵ درجه فراتر رفت.
    *   اگر ظرفیت دیسک به ۹۰٪ رسید.
    *   اگر نود از مش (Mesh) جدا شد.

---

## ۵. پشتیبان‌گیری و بازیابی (Backup)

*   **TiDB**: به صورت خودکار از ابزار `BR (Backup & Restore)` برای تهیه پشتیبان در بازه‌های ۱۲ ساعته روی S3 یا MinIO استفاده می‌شود.
*   **IPFS**: داده‌های حیاتی (مثل Wasm Agents) باید در چندین نود "Pin" شوند.
*   **دستور پشتیبان‌گیری دستی**:
    ```bash
    ./nexus-cli backup --all --output ./backups/latest.tar.gz
    ```

---

## ۶. پاداش‌ها و جریمه‌ها (Slashing)

به عنوان اپراتور، شما بر اساس **Uptime** و **Compute Accuracy** پاداش می‌گیرید.
*   **جریمه (Slash)**: اگر نود شما در حین انجام یک کار محاسباتی (Inference) خاموش شود یا نتیجه اشتباه برگرداند، بخشی از توکن‌های استیک شده شما کسر خواهد شد.
